info 概述

SSH Tunnel Manager 是一个基于 Docker 的解决方案，使用 autossh 和 YAML 配置文件来管理 SSH 隧道。

主要功能：

  • Docker化：环境封装，易于部署和管理
  • 网页配置：通过可视化界面管理隧道配置
  • 自动重载：检测配置变化并自动重载服务
  • 双向隧道：支持本地到远程和远程到本地的隧道
  • 多架构支持：支持 AMD64、ARM64 等多种架构

rocket_launch 快速设置

1. 下载配置文件

mkdir autossh-tunnel
cd autossh-tunnel

# 下载 docker-compose.yaml
curl -O https://oaklight.github.io/autossh-tunnel-dockerized/compose.yaml

# 创建配置目录
mkdir config
touch config/config.yaml

2. 配置用户权限

# 查看当前用户 UID/GID
id

# 设置环境变量
export PUID=$(id -u)
export PGID=$(id -g)

3. 启动服务

docker compose up -d

4. 访问网页界面

打开浏览器访问：http://localhost:5000

web 网页配置

界面说明：

  • Name：隧道名称，用于标识
  • Status：隧道当前状态（运行/停止/错误）
  • Remote Host：远程主机地址（如：user@server.com）
  • Remote Port：远程端口或主机:端口格式
  • Local Port：本地端口或IP:端口格式
  • Direction：隧道方向（远程到本地/本地到远程）
  • Actions：操作按钮（交互认证/删除）

交互认证：

点击指纹图标可以切换交互认证状态：

  • 🫆 绿色：启用交互认证
  • 🫆 灰色：禁用交互认证

保存配置：

点击 Save & Restart 按钮保存配置并重启隧道服务。

隧道详情页面：

点击主页面表格中的隧道名称可以打开该隧道的详情页面。

详情页面提供以下功能：

  • 查看隧道的完整配置信息
  • 启动、重启或停止隧道
  • 实时查看隧道日志
  • 监控隧道运行状态

description YAML配置

基本格式：

tunnels:
  # 将本地服务暴露到远程服务器
  - name: "web-service"
    remote_host: "user@remote-host1"
    remote_port: 22323
    local_port: 18120
    direction: local_to_remote
    interactive: false
    
  # 将远程服务映射到本地端口
  - name: "database"
    remote_host: "user@remote-host2"
    remote_port: 8000
    local_port: 8001
    direction: remote_to_local
    interactive: true

高级配置 - 指定绑定地址：

tunnels:
  # 指定远程绑定地址
  - remote_host: "user@server"
    remote_port: "192.168.45.130:22323"
    local_port: 18120
    direction: local_to_remote
    
  # 指定本地绑定地址
  - remote_host: "user@server"
    remote_port: 22323
    local_port: "192.168.1.100:18120"
    direction: local_to_remote

vpn_key SSH配置

重要：本项目严重依赖 ~/.ssh/config 文件进行 SSH 连接配置。

SSH配置文件示例 (~/.ssh/config)：

Host myserver
    HostName 192.168.1.100
    User myuser
    Port 22
    IdentityFile ~/.ssh/id_ed25519
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null

Host production
    HostName prod.example.com
    User deploy
    Port 2222
    IdentityFile ~/.ssh/prod_key
    ProxyJump bastion

SSH密钥权限：

chmod 700 ~/.ssh
chmod 600 ~/.ssh/*
chmod 644 ~/.ssh/*.pub

terminal 交互认证命令行

对于需要双因素认证（2FA）或密码认证的 SSH 服务器，您需要使用 autossh-cli 命令行工具进行交互式认证。

何时使用：

  • 服务器启用了双因素认证（2FA/MFA）
  • 服务器需要密码认证而非密钥认证
  • 服务器使用键盘交互式认证

使用方法：

按照以下步骤启动交互式隧道：

  1. 在网页界面启用隧道的交互认证选项
  2. 保存配置后，获取隧道的哈希值
  3. 使用以下命令启动交互式隧道

命令示例：

使用 docker compose（推荐）：

docker compose exec -it -u myuser autossh autossh-cli auth <hash>

使用 docker exec（需要容器名称）：

docker exec -it -u myuser autossh-tunnel-autossh-1 autossh-cli auth <hash>

示例（使用 8 位哈希前缀）： 

docker compose exec -it -u myuser autossh autossh-cli auth 7b840f83

哈希前缀支持：

类似于 Git 的短哈希，您可以使用 8 个或更多字符的哈希前缀来标识隧道。

其他常用命令：

列出所有配置的隧道：

docker compose exec -u myuser autossh autossh-cli list

查看隧道状态：

docker compose exec -u myuser autossh autossh-cli status

查看特定隧道的日志：

docker compose exec -u myuser autossh autossh-cli logs <hash>

停止特定隧道：

docker compose exec -u myuser autossh autossh-cli stop-tunnel <hash>

注意事项：

  • 必须使用 -it 参数以启用交互式终端
  • 使用 -u myuser 指定正确的用户身份
  • 交互式隧道会在前台运行，认证成功后会自动转入后台
  • 启用交互认证的隧道暂时无法通过网页界面启动，必须使用命令行

settings 高级配置

环境变量：

  • PUID：容器用户 UID（默认：1000）
  • PGID：容器用户 GID（默认：1000）

端口格式：

  • 简单端口：8080
  • 主机:端口：hostname:8080
  • IP:端口：192.168.1.100:8080

隧道方向：

  • remote_to_local：将远程服务映射到本地（默认）
  • local_to_remote：将本地服务暴露到远程

安全注意事项：

启用远程端口转发时，可能需要在远程服务器配置 GatewayPorts：

# 编辑 /etc/ssh/sshd_config
GatewayPorts clientspecified

# 重启 SSH 服务
sudo systemctl restart sshd

bug_report 故障排除

常见问题：

🔴 隧道连接失败
  • 检查 SSH 配置文件 ~/.ssh/config
  • 验证 SSH 密钥权限
  • 确认远程主机可访问
🔴 权限错误
  • 检查 PUID/PGID 设置
  • 确认 SSH 密钥文件权限
  • 验证 Docker 用户组权限
🔴 配置不生效
  • 检查 YAML 语法是否正确
  • 确认配置文件路径
  • 重启容器服务

查看日志：

# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker compose logs -f autossh
docker compose logs -f web

重启服务：

# 重启所有服务
docker compose restart

# 重启特定服务
docker compose restart autossh
