info Overview

SSH Tunnel Manager is a Docker-based solution for managing SSH tunnels using autossh and YAML configuration files.

Key Features:

  • Dockerized: Environment encapsulation, easy to deploy and manage
  • Web Config: Visual interface for managing tunnel configurations
  • Auto Reload: Detects configuration changes and automatically reloads services
  • Bidirectional Tunnels: Supports both local-to-remote and remote-to-local tunnels
  • Multi-arch Support: Supports AMD64, ARM64 and more architectures

rocket_launch Quick Setup

1. Download Configuration Files

mkdir autossh-tunnel
cd autossh-tunnel

# Download docker-compose.yaml
curl -O https://oaklight.github.io/autossh-tunnel-dockerized/compose.yaml

# Create config directory
mkdir config
touch config/config.yaml

2. Configure User Permissions

# Check current user UID/GID
id

# Set environment variables
export PUID=$(id -u)
export PGID=$(id -g)

3. Start Services

docker compose up -d

4. Access Web Interface

Open browser and visit: http://localhost:5000

web Web Configuration

Interface Description:

  • Name: Tunnel name for identification
  • Status: Current tunnel status (Running/Stopped/Error)
  • Remote Host: Remote host address (e.g., user@server.com)
  • Remote Port: Remote port or host:port format
  • Local Port: Local port or IP:port format
  • Direction: Tunnel direction (Remote to Local / Local to Remote)
  • Actions: Action buttons (Interactive Auth / Delete)

Interactive Auth:

Click the fingerprint icon to toggle interactive auth status:

  • Green: Interactive auth enabled
  • Gray: Interactive auth disabled

Save Configuration:

Click the Save & Restart button to save configuration and restart tunnel services.

Tunnel Detail Page:

Click the tunnel name in the main page table to open the tunnel detail page.

The detail page provides the following features:

  • View complete tunnel configuration
  • Start, restart, or stop the tunnel
  • View real-time tunnel logs
  • Monitor tunnel running status

description YAML Configuration

Basic Format:

tunnels:
  # Expose local service to remote server
  - name: "web-service"
    remote_host: "user@remote-host1"
    remote_port: 22323
    local_port: 18120
    direction: local_to_remote
    interactive: false

  # Map remote service to local port
  - name: "database"
    remote_host: "user@remote-host2"
    remote_port: 8000
    local_port: 8001
    direction: remote_to_local
    interactive: true

Advanced - Specify Bind Address:

tunnels:
  # Specify remote bind address
  - remote_host: "user@server"
    remote_port: "192.168.45.130:22323"
    local_port: 18120
    direction: local_to_remote

  # Specify local bind address
  - remote_host: "user@server"
    remote_port: 22323
    local_port: "192.168.1.100:18120"
    direction: local_to_remote

vpn_key SSH Configuration

Important: This project relies heavily on ~/.ssh/config for SSH connection configuration.

SSH Config Example (~/.ssh/config):

Host myserver
    HostName 192.168.1.100
    User myuser
    Port 22
    IdentityFile ~/.ssh/id_ed25519
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null

Host production
    HostName prod.example.com
    User deploy
    Port 2222
    IdentityFile ~/.ssh/prod_key
    ProxyJump bastion

SSH Key Permissions:

chmod 700 ~/.ssh
chmod 600 ~/.ssh/*
chmod 644 ~/.ssh/*.pub

terminal Interactive Auth CLI

For SSH servers requiring 2FA or password authentication, use the autossh-cli command line tool for interactive authentication.

When to Use:

  • Server has 2FA/MFA enabled
  • Server requires password authentication instead of key-based
  • Server uses keyboard-interactive authentication

Usage:

Follow these steps to start an interactive tunnel:

  1. Enable interactive auth in the web interface
  2. After saving config, get the tunnel hash
  3. Use the following command to start the interactive tunnel

Command Examples:

Using docker compose (recommended):

docker compose exec -it -u myuser autossh autossh-cli auth <hash>

Using docker exec (requires container name):

docker exec -it -u myuser autossh-tunnel-autossh-1 autossh-cli auth <hash>

Example (using 8-char hash prefix):

docker compose exec -it -u myuser autossh autossh-cli auth 7b840f83

Hash Prefix Support:

Similar to Git short hashes, you can use 8 or more characters of the hash prefix to identify a tunnel.

Other Common Commands:

List all configured tunnels:

docker compose exec -u myuser autossh autossh-cli list

View tunnel status:

docker compose exec -u myuser autossh autossh-cli status

View logs for a specific tunnel:

docker compose exec -u myuser autossh autossh-cli logs <hash>

Stop a specific tunnel:

docker compose exec -u myuser autossh autossh-cli stop-tunnel <hash>

Notes:

  • Must use -it flag to enable interactive terminal
  • Use -u myuser to specify the correct user identity
  • Interactive tunnel runs in foreground, automatically moves to background after authentication
  • Tunnels with interactive auth cannot be started via web interface, must use CLI

settings Advanced Configuration

Environment Variables:

  • PUID: Container user UID (default: 1000)
  • PGID: Container user GID (default: 1000)

Port Formats:

  • Simple port: 8080
  • Host:port: hostname:8080
  • IP:port: 192.168.1.100:8080

Tunnel Directions:

  • remote_to_local: Map remote service to local (default)
  • local_to_remote: Expose local service to remote

Security Notes:

When enabling remote port forwarding, you may need to configure GatewayPorts on the remote server:

# Edit /etc/ssh/sshd_config
GatewayPorts clientspecified

# Restart SSH service
sudo systemctl restart sshd

bug_report Troubleshooting

Common Issues:

Tunnel Connection Failed
  • Check SSH config file ~/.ssh/config
  • Verify SSH key permissions
  • Confirm remote host is accessible
Permission Error
  • Check PUID/PGID settings
  • Confirm SSH key file permissions
  • Verify Docker user group permissions
Configuration Not Taking Effect
  • Check YAML syntax
  • Confirm config file path
  • Restart container services

View Logs:

# View all service logs
docker compose logs -f

# View specific service logs
docker compose logs -f autossh
docker compose logs -f web

Restart Services:

# Restart all services
docker compose restart

# Restart specific service
docker compose restart autossh
